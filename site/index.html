GDPevo: The First Benchmark for
Agent Self-Evolution on GDP-Valued Tasks GDPevo：首个评估 Agent
自进化能力的 GDP-Valued 基准

GDPevo evaluates whether agents can self-evolve on economically valuable (GDP) work — get better at real industry tasks through their own experience. In each business environment, an agent works through related train tasks, turns that experience into reusable skills, and is then evaluated on held-out tasks. Like GDPval, the work is real and economically meaningful; unlike it, GDPevo is stateful, so experience carries from one task to the next. GDPevo 评估 agent 能否在具有经济价值（GDP）的任务上自进化 —— 即通过自身经验，把真实产业工作做得越来越好。 在每个业务环境中，agent 先完成一组相关的 train tasks，把经验沉淀为可复用的 skill，再在 held-out tasks 上接受评估。与 GDPval 一样， 这些工作真实且具有经济价值；不同的是，GDPevo 是有状态的 —— 经验会从一个任务延续到下一个。

Abstract摘要

Most benchmarks can't measure self-evolution. 多数基准无法衡量自进化。

Benchmarks for self-evolving agents are still scarce, and most existing benchmarks can only evaluate stateless agent systems: every task starts from a blank slate, and nothing an agent learns on one task is allowed to help the next. Yet the value of an agent in real economic work comes precisely from the opposite — from accumulating experience and getting faster and more accurate over time. 专门评估自进化 agent 的基准仍然十分稀缺，绝大多数现有基准只能评估无状态的 agent 系统：每个任务都从零开始， agent 在一个任务上学到的东西无法帮助下一个任务。但 agent 在真实经济工作中的价值，恰恰来自相反的方向 —— 来自经验的累积， 以及随时间变得更快、更准。

GDPevo closes this gap. The benchmark's unit is the task group: one shared business environment, five train tasks, and five held-out test tasks that share a family of transferable procedures. Train tasks are the experience source; held-out test tasks measure whether the agent self-evolves — whether experience becomes a portable operating method. Across the first release of 120 tasks in 12 groups, self-evolving agents improve held-out accuracy by 18.21 percentage points after inductive learning, while reducing solver token cost by 25.75% — agents do not just get better, they get better while doing less work. GDPevo 填补了这一空白。基准的基本单元是 task group：一个共享业务环境、5 个 train tasks，以及 5 个共享同一类 可迁移流程的 held-out test tasks。Train tasks 是经验来源，held-out test tasks 衡量 agent 是否自进化 —— 即经验是否真正变成可迁移的操作方法。在首个版本的 12 组共 120 个任务上，自进化 agent 在归纳学习后把 held-out 准确率平均提升 18.21 个百分点，同时把 solver 的 token 成本削减 25.75% —— agent 不只是变强了，而是在做更少工作的同时变强。

Evolution methods进化方法

Two ways to self-evolve vs. a non-evolving agent. 两种自进化方法，对比未进化的 Agent。

We compare a non-evolving baseline with two self-evolution methods. The same solver, harness, and evaluators run across all three conditions — only the evolved skill context changes. 我们用一个未进化的基线，对比两种自进化方法。三种条件使用完全相同的 solver、harness 与评测器 —— 唯一变化的是进化得到的 skill 上下文。

no_skill

Non-evolving baseline未进化基线

The agent solves each held-out test task from scratch, with no memory of related work — it never evolves.agent 从零解决每个 held-out test task，对相关工作没有任何记忆 —— 完全不进化。

demonstration_skill

Demonstration-based evolution基于示范的进化

The agent self-evolves by inducing skills from worked demonstrations of the train tasks, then applying them at test time.agent 研究 train tasks 的完整示范，从中归纳出 skill，并在测试时应用 —— 以此实现自进化。

reflection_skill

Reflection-based evolution基于反思的进化

The agent self-evolves by attempting train tasks, reflecting on its own outcomes, and distilling skills from that reflection.agent 先尝试 train tasks，再对自己的结果进行反思，从反思中提炼出 skill —— 以此实现自进化。
