本任务的业务背景是法官完成 sentencing docket 后，书记员需要把庭审 notes、portal 记录、过期 fee worksheet 和 source conflict 统一为最终 disposition entry、financial entry 和 register totals。这与源例子中的刑事判决后处理流程一致，但案件、县、金额和冲突事实都是新的。

评测包含 8 个 whole-point scoring checks，原始权重为 `[1, 2, 3, 3, 3, 1, 3, 1]`。`SP001` 检查 included/excluded matter；`SP002` 检查 DOB、criminal-history 和 counsel/source conflict 的 audit resolution；`SP003` 检查 charge disposition、plea、counsel、criminal-history 和 sentence；`SP004` 检查逐案 financial entries；`SP005` 检查旧 worksheet trap、unsupported fee、pending draft amount 和 waived fine；`SP006` 检查 docket entries；`SP007` 检查 register totals；`SP008` 检查 controlled source-basis fields。这些分数覆盖 matter selection、audit/source resolution、legal disposition、financial calculation、fee reconciliation、docket posting、register aggregation 和 source classification，避免重复奖励同一个事实。较高权重集中在 base attempts 更常遗漏、且依赖迁移的 source precedence、stale-fee 和 aggregate closeout 决策上。
